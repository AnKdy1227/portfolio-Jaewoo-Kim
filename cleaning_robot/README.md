# 🤖 STM32-Based Autonomous Cleaning Robot

## 📖 개요
- **기간**: 2024.06 ~ 2024.09
- **기술 스택**: STM32 (ARM Cortex-M, HAL Driver), C, 초음파 센서(HC-SR04) ×4, DC 모터 ×4, OLED(SSD1306), PS2 컨트롤러, 조도 센서, 부저
- **역할**: 팀원 / MCU 펌웨어 개발 담당
- **개발 환경**: STM32CubeIDE, STM32CubeMX, HAL Driver

이 프로젝트는 **자율주행 + 먼지 흡입 + 경로 복귀 기능을 통합한 스마트 청소 로봇**입니다.
PS2 컨트롤러로 수동 조종이 가능하며, 자율주행 모드에서는 4방향 초음파 센서를 기반으로 장애물을
회피하고, 어느 시점에서든 복귀 모드를 활성화하면 출발 지점까지 자동 복귀합니다.

## 💡 주요 구현 기능

### 🚗 통합 제어 펌웨어
- **자율주행 / 수동 조종 / 경로 복귀 / 경고 모드**를 하나의 펌웨어에서 통합 관리
- `robot_mode_t` enum으로 모드를 추상화, 메인 루프의 `switch` 분기로 모드별 동작 수행
- PS2 버튼 엣지 검출로 모드 전환 (SELECT: 자율 ↔ 수동 / L1: 복귀 시작 / START: 경고음 토글)

### 🧭 상태 머신 기반 자율주행
- **6개 주행 상태**로 정의: `STATE_FORWARD / SLOW_FORWARD / TURN_LEFT / TURN_RIGHT / BACKWARD / STOP`
- 한 번 진입한 회피 상태는 **최소 유지 시간(`STATE_HOLD_TURN_MS` 등)** 동안 유지 → 짧은 주기로
  좌우가 반복 전환되어 제자리에서 흔들리는 문제 차단
- 거리 임계값을 **경고(`DIST_WARNING_CM`)와 위험(`DIST_DANGER_CM`) 두 단계로 분리**
  - 경고 거리 → `SLOW_FORWARD` 로 감속 전진
  - 위험 거리 → `STOP` 으로 관성 제거 후 회피 동작 진입

### 🛡️ 안전한 회피 로직
- **후진 동작은 후방 센서값(`DIST_REAR_SAFE_CM`)을 선확인** 한 뒤에만 수행 → 앞이 막힌 상황에서
  뒤쪽 충돌 위험을 함께 다룸
- 좌우 회피 방향은 **좌우 센서값을 비교해 더 넓은 공간 쪽으로 회전** → 같은 상황에서 회전 방향이
  일관되게 결정됨
- 좌·우·후방이 모두 막히면 `STOP` 상태로 떨어져 다음 사이클에 재판단

### 📊 초음파 센서 신호 처리
- 한 번의 측정으로 즉시 판단하지 않고, **연속 3회 측정 후 평균값**을 사용
- 측정 실패값(타임아웃, 범위 초과 등)은 **평균에서 제외** → 잡음에 의한 오판단을 줄이면서
  유효 샘플만으로 안정적인 거리값 산출

### ↩️ 경로 기록 및 복귀
- 자율주행 중 수행한 명령(전진/후진/좌회전/우회전)을 `path_log_t.buf[]` 에 **순서대로 기록**
- 복귀 모드 진입 시 기록된 명령을 **역순으로 꺼내 반대 명령으로 변환** 후 실행
  - `FORWARD ↔ BACKWARD`, `TURN_LEFT ↔ TURN_RIGHT`
- 한 명령당 충분한 유지 시간을 보장한 뒤 다음 명령으로 진행
- 인덱스가 0이 되면 자동으로 정지 후 수동 모드로 복귀

### 📺 OLED 상태 모니터링
- 현재 **모드 / 상태 / 4방향 거리 / 경로 버퍼 사용량 / 브러시 상태**를 실시간 표시
- SSD1306 I2C 드라이버 사용

### 🔔 경고 모드
- START 버튼 토글로 활성화
- 4방향 중 어느 쪽이라도 위험 거리 진입 시 부저 ON

### 🧹 청소 기능
- 브러시 모터는 별도 PWM 채널(`TIM3 CH1`)로 제어, 모드와 독립적으로 ON/OFF 가능
- 중앙 쿨러로 먼지 흡입, 내부 더스트박스에 수집

## 🛠 하드웨어 구성
| 구분          | 부품                                                      |
| ------------- | --------------------------------------------------------- |
| MCU           | STM32F4xx (ARM Cortex-M4)                                 |
| 센서          | HC-SR04 초음파 센서 ×4 (전 / 후 / 좌 / 우), CdS 조도 센서 |
| 구동부        | DC 모터 ×4 (주행 ×2, 브러시 ×2), 쿨러 팬 ×1               |
| 모터 드라이버 | L298N                                                     |
| 출력부        | SSD1306 OLED (I2C), 부저, LED                             |
| 입력부        | PS2 컨트롤러 (수동 모드)                                  |

## 🧩 코드 구조

```
cleaning_robot_stm32.c
├─ ① 매크로 / 상수 정의 (거리 임계값, 상태 유지 시간, PWM duty)
├─ ② 타입 정의 (drive_state_t, robot_mode_t, distance_t, path_log_t)
├─ ③ 전역 변수 (현재 상태, 모드, 경로 버퍼)
├─ ④ 저수준 유틸 (us 단위 타이머, busy-wait delay)
├─ ⑤ 초음파 측정 (1회 측정 + 3회 평균 필터)
├─ ⑥ 모터 제어 (PWM duty 기반 좌/우 모터, 브러시)
├─ ⑦ 상태 머신 (상태 진입, 출력 매핑, hold 판정, 경로 기록)
├─ ⑧ 자율주행 (거리 2단계 분기 + 좌우 비교 + 후방 선확인)
├─ ⑨ 경로 복귀 (역순 + 반대 명령)
├─ ⑩ 수동 모드 (PS2 입력 → 상태 전이)
├─ ⑪ 경고 모드 (위험 거리 진입 시 부저)
├─ ⑫ OLED 출력 (모드 / 상태 / 거리 / 경로 길이)
├─ ⑬ PS2 버튼 모드 전환 (엣지 검출)
└─ ⑭ main loop : 50Hz 통합 제어 루프
```

## ⚙️ 제어 루프 동작 순서 (메인 루프 1 사이클)

```
┌─ PS2 입력 + 4방향 거리 측정 (3회 평균)
├─ 모드 전환 입력 처리 (SELECT / L1 / START)
├─ 모드별 동작
│   ├─ MODE_AUTO    → autonomous_step()  ─┐
│   ├─ MODE_RETURN  → return_step()       ├─ 상태 머신을 통한 모터 PWM 갱신
│   └─ MODE_MANUAL  → manual_step()       ─┘
├─ 경고 부저 갱신
├─ OLED 화면 갱신
└─ 20ms 대기 (≈ 50Hz)
```

## 🚀 실행 / 사용 방법
1. **CubeMX 프로젝트 설정**
   - TIM2 PWM 4채널 (좌/우 모터 H-bridge 제어)
   - TIM3 PWM 1채널 (브러시 모터)
   - TIM5 us 단위 카운터 (초음파 echo 폭 측정용)
   - I2C1 (OLED), GPIO (초음파 Trig/Echo, PS2, 부저)
2. **펌웨어 빌드 및 플래시**
   - STM32CubeIDE에서 `cleaning_robot.c` 를 프로젝트에 추가하여 빌드
   - ST-Link로 보드에 플래시
3. **동작 모드 선택**
   - 기본은 수동 모드. **SELECT** 로 자율주행 ON/OFF
   - 자율주행 후 **L1** 으로 출발 지점 복귀
   - **START** 로 근접 경고 부저 ON/OFF

## ✨ 느낀 점
- **상태 머신 설계 경험**: 매 루프마다 즉시 명령을 결정하는 구조 대신 상태를 자료구조로 명시하고
  전이 조건을 분리하는 방식이 시스템 안정성에 어떻게 기여하는지 체감했습니다.
- **신호 전처리의 가치**: 평균 필터 + 실패값 제외라는 단순한 처리만으로도 노이즈가 시스템에
  미치는 영향을 크게 줄일 수 있음을 확인했습니다.
- **방어적 설계 사고**: "앞을 피하기 위해 뒤로 가는데 뒤도 막혀 있으면?" 같은 코너 케이스를
  명시적으로 처리하는 습관이 신뢰성 있는 임베디드 SW의 시작이라는 점을 배웠습니다.
- **HAL 기반 임베디드 개발 경험**: PWM, GPIO, I2C, 타이머를 직접 다루며 MCU 주변 장치의
  저수준 동작을 이해할 수 있었습니다.
